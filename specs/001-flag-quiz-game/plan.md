# Implementation Plan: Vexillologist — Flag Quiz Game

**Branch**: `001-flag-quiz-game` | **Date**: 2026-04-28 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/001-flag-quiz-game/spec.md`

## Summary

Build a country-flag guessing game (SvelteKit 2 + Svelte 5) where players identify a
flag from 4 multiple-choice options within 15 seconds. Scoring rewards speed and streaks.
Country data is fetched once from REST Countries API and cached in localStorage for
offline play. Scores are persisted locally (top-10). Drizzle + Turso + Lucia are
scaffolded for future auth and cloud score sync.

## Technical Context

**Language/Version**: TypeScript 5.x — `strict: true` throughout
**Primary Dependencies**: SvelteKit 2, Svelte 5, Tailwind CSS 4, Drizzle ORM, Turso
  (LibSQL), Lucia v3, Vite+ check/test
**Storage**: localStorage (active in MVP); Turso/LibSQL via Drizzle (auth scaffold only)
**Testing**: Vitest via `viteplus test`
**Target Platform**: Web — modern browsers (Chrome, Firefox, Safari, Edge, current + 1 prior)
**Project Type**: Single-page web application (SvelteKit file-based routing)
**Performance Goals**: Zero network latency during active gameplay; initial load < 3 s on
  fast 3G (flag data ~70 KB)
**Constraints**: Offline-capable after first load; 320 px+ viewport; WCAG AA; 44 px touch
  targets
**Scale/Scope**: Single-user, anonymous in MVP; ~250 countries dataset

## Phase 0: Research

**Key decisions**:
- Scaffold: `npx sv create .` (SvelteKit skeleton + TypeScript + Vitest)
- Tailwind v4: `@tailwindcss/vite` plugin (no `tailwind.config.js`)
- Vite+ check/test wired to `npm run check` and `npm run test`

- Lucia v3: usable for MVP; flagged for post-MVP migration to `better-auth`

## Phase 1: Design

See [data-model.md](data-model.md), [contracts/](contracts/), [quickstart.md](quickstart.md).

**Component tree**:

```
+layout.svelte
  └── slot
       ├── +page.svelte (/)           → HomeScreen
       │     ├── LeaderBoard.svelte
       │     └── PlayButton.svelte
       └── game/+page.svelte (/game)  → GameScreen
             ├── FlagImage.svelte     → <img> with alt, aspect-ratio, lazy=false
             ├── Timer.svelte         → countdown bar + label, urgency at <5 s
             ├── ChoiceGrid.svelte    → 2×2 grid of AnswerButton (keys 1–4)
             ├── StreakBadge.svelte   → multiplier indicator
             └── ScoreBar.svelte      → running score + flag counter
```

**Key implementation notes**:
- `game/+page.ts` (`load` function) reads country cache before the route renders —
  prevents flash of loading state on return visits
- `FlagImage.svelte` wraps `<img>` with an `onerror` handler that calls `replaceFlagOnError`
  on the game store; the replacement is silent to the player
- `Timer.svelte` uses `$effect` to subscribe to `timeLeft` from the game store; applies
  Tailwind classes `text-red-600` + aria-label `"Hurry!"` when `timeLeft < 5000`
- All answer buttons have `aria-label="Option {n}: {countryName}"` and respond to
  keydown events for digits 1–4 at the document level (not per-button)
- Feedback phase: the game store transitions automatically via `setTimeout(1500)`; the
  correct answer button turns green, the selected wrong answer (if any) turns red
